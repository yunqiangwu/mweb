HZERO全新脚手架HZERO-CLI不容错过！

Posted 2020/02/10

全新的技术架构！

基于 QIANKUN & NEST-CLI & LERNA 升级了整体架构。

我们把一个前端项目分成以下几个部分：

  • hzero-boot: 定义了一组 api 实现 页面整体的布局、全局配置、规范子模块的页面路由组合方式等、被父工程和子工程依赖，父工程也可以直接当 hzero-boot。
  • hzero-webpack-script: 封装 webpack 构建的配置和 webpack 脚本。
  • hzero-cli: 命令行工具，这里可以封装一些处理代码的小工具，比如代码的 hzero 多语言处理工具等。
  • hzero-schemas: 这里是用来页面可以模块的模板，模块可以是基于 react 或者 vue。
  • 微前端子模块: 基于组件化的思想， 子模块也是可以一个组成部分。

上面的每个部分也可组装可替换。

比如我们的视图布局想要换成 aliyun console 的样式，我们只需要重新开发一个 hzero-boot-aliyun-console 把它替换掉 hzero-boot 就可以了。

丰富的功能

  1. 代码多语言扫描
  2. 支持增量编译
  3. lerna 的多个子模块管理
  4. 代码生成器
  5. typescript 支持

解决了 HZERO 实施当中一系列痛点问题！

  1. 文件过多导致 webpack 编译的内存溢出。
  2. 不必要构建时间增加：每次构建都需要全部文件扫描编译，比如在一千个文件中，只修改了其中一个文件一行代码，每次重新发布版本时，也需要全部编译一次。
  3. 多团队并行开发时合并代码容易冲突。

话不多说，我们如何进行操作使用呢？

下面将示范一下 hzero-cli 的简单使用流程

安装 hzero-cli

注意：hzero-cli 需要安装 node 10 以上的版本，这里建议大家安装 node 最新稳定版本(lts 版)。

全局安装 hzero-cli 命令行工具

npm install -g hzero-cli --registry http://nexus.saas.hand-china.com/content/groups/hzero-npm-group # 安装 hzero-cli 工具

生成项目

hzero-cli new sample # 创建 hzero 前端项目
cd sample && hzero-cli info # 查看当前工程信息

创建子模块

npx hzero-cli g sub-module test-module  # 创建一个名字为 test-module 的子模块

运行完成之后，当前项目目录会多一个文件夹 packages/sample-test-module, 这里 hzero 中规定所有子模块的名字前缀默认都是父工程的名字。

创建页面

进入子模块目录：

cd packages/sample-test-module

在子模块下创建一个页面：

npx hzero-cli g simple-page test-page # 创建一个名字为 test-page 的页面

创建完成生成一个简单页面,生成 packages/sample-test-module/src/routes/test-page 文件夹。

启动调试

yarn start

开发调试可以可以在父项目目录运行，也可以在子模块目录运行。
区别在于:

  • 父项目目录运行调试只是调试子模块的加载情况，调试的是编译后的子模块数据，如果想调试那个子模块，先用按需增量编译那个子模块
  • 子模块目录运行 调试，调试的范围是只是当前子模块下的页面

当开始编译后会自动打开浏览器，通过 localhost:8000，查看页面效果。

全量编译

进入父工程目录

yarn run build

全量编译会删掉之前编译的所有代码。
全量编译会生成 dist 文件夹， 所有子模块的代码会编译到 dist/packages下以子模块的名字命名的文件夹。

增量编译

进入父工程目录

yarn run build:ms sample-test-module # 增量编译名字为 sample-test-module 的模块

注意：增量编译时， 如果父工程已经编译过， 这里不会重新编译父项目文件。
增量编译重新生成 dist/packages下以子模块的名字命名的文件夹，以及 packages 文件下的 microConfig.json 索引文件。
